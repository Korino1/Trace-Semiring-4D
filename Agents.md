## Политика контекста для субагентов

Оркестратор держит в своей памяти полный концепт проекта, инварианты  и весь рабочий профиль. Это не должно расползаться по окну контекста на каждой мелкой подзадаче.

Для каждого НОВОГО субагента действует такой порядок:

- он один раз получает полный canonical packet;
- canonical packet обязательно включает документацию, концепт и ключевые исходники;
- после этого субагент работает в своей линии задач, опираясь на уже полученный контекст;
- поверх canonical packet он получает узкий ownership, критерии успеха и жёсткий запрет на упрощения, заглушки и имитации.


Harness-протокол правок (обязательное): КАЖДОМУ субагенту показывать этот протокол!
- Перед любой правкой кода/документа сначала выполняй `scripts/harness_read.ps1` (read-range) и фиксируй `range_hash`.
- Применяй правки только через `scripts/harness_apply.ps1` с `expected_hash`; «слепые» замены без hash-check запрещены.
- При несовпадении хеша (`stale edit`) остановись, перечитай диапазон и пересобери правку;
- В спецификации правок используй только repo-relative пути (без локальных абсолютных путей пользователя).
